# خارطة تحويل «متن» إلى قارئ Markdown تنافسي

## الخلاصة الاستراتيجية

- لا يتحول «متن» إلى محرر. Obsidian يدعم RTL رسميًا واكتشاف اتجاه السطر وواجهة عربية معكوسة؛ منافسته كمحرر ستستهلك المشروع في المؤشر والتحديد والتراجع واللصق والتنسيق دون قيمة فريدة كافية.
- لا يُضاف AI في المرحلة الحالية؛ لا توجد مشكلة قراءة محددة تبرر التكلفة والتعقيد، ولن يكون التلخيص العام ميزة تنافسية أمام Notion.
- التموضع المقترح: **طبقة قراءة احترافية فوق ملفاتك الحالية**؛ تفتح مجلد Obsidian أو مستودع Git أو مجموعة ملفات عادية، دون استيراد أو حساب أو تعديل محتوى الملفات.
- الميزة الأساسية: **اقرأ، تنقّل، استأنف، علّق، ثم افتح الملف في محررك المفضل؛ ولا يغيّر «متن» محتوى Markdown، ولا يدير الملف نفسه إلا بإجراء صريح وقابل للاستعادة من المستخدم**.
- جودة العرض وحدها لم تعد كافية؛ يجب أن يتميز «متن» بذاكرة القراءة، والبحث العربي، والتعليقات الجانبية، وسهولة الوصول بين الملفات.

| المنتج | مهمته الأساسية |
|---|---|
| Obsidian | إنشاء المعرفة وربطها وتحريرها وتوسعتها بالإضافات |
| Notion | إنشاء مساحات عمل وقواعد بيانات وتعاون |
| Word | تحرير مستندات Office وتبادلها |
| متن | قراءة ملفات Markdown المحلية بتركيز وجودة، مع إدارة محدودة وصريحة لدورة حياة الملف |

## مراحل التنفيذ

### 1. تثبيت الأساس قبل إضافة المزايا

- [x] تنفيذ خطط الإصلاح الحالية `001–005`: حماية العرض، واستخراج نواة التصيير، واختبارات XSS والاتجاه والرياضيات، والتحقق من بناء `docs`.
- [x] معالجة أخطاء الخادم الموثقة: فشل `listen`، وتراكم مراقبي الملفات، ومسارات العرض غير المحمية، وإغلاق اتصالات SSE بأمان.
- [x] إضافة تحقق `Host` وCSP ورفض طرق HTTP غير المدعومة.
- [x] إضافة تحقق `Origin` ورمز جلسة عشوائي بالتزامن مع أول API يكتب حالة أو يشغّل محررًا خارجيًا؛ تنفذ `PUT /api/state` كتابة ذرية خارج مجلد المستخدم وترفض الطلبات غير المطابقة.
- [x] إبقاء الخادم بلا تبعيات npm تشغيلية، مع إضافة اختبارات تكامل للمتصفح عبر Chrome DevTools والاختبارات الآلية الحالية.
- نشر الحزمة لاحقًا باسم scoped مثل `@ajarallah/matn` مع استمرار الأمر التنفيذي `matn`؛ الاسم العام `matn` مستخدم حاليًا لحزمة أخرى.

### 2. بناء العمود الفقري للتنقّل

- [x] إضافة شاشة بداية تعرض «متابعة القراءة»، والمجلدات والملفات الأخيرة. عند فتح مجلد: يستأنف آخر ملف، ثم يفضّل `README.md`، ثم أول ملف أبجديًا.
- [x] إضافة مبدّل ملفات سريع عبر `Cmd/Ctrl+K`، يعرض الملفات الأخيرة ويبحث بالاسم والمسار والاسم البديل في YAML.
- [x] إضافة بحث شامل في المجلد عبر `Cmd/Ctrl+Shift+F` مع مقتطفات ونتائج قابلة للتنقل؛ البحث العربي يتجاهل التشكيل والتطويل، ويوحّد أشكال الألف و`ى/ي` دون دمج `ة/ه`.
- [x] تحويل `[[wikilinks]]` والروابط النسبية إلى روابط عاملة بين الملفات، مع معالجة الأسماء البديلة والعناوين الداخلية.
- [x] إضافة الرجوع والتقدّم، ومعاينة الرابط عند المرور، والروابط الواردة مع موضع وسياق كل إشارة.
- [x] إضافة قائمة إجراءات للملف النشط وملفات الشجرة: «فتح في المحرر»، و«إظهار في المجلد»، و«نسخ المسار»، و«نقل إلى سلة المهملات». لا يوضع زر حذف مباشر في الشريط العلوي.
- [x] تنفيذ النقل إلى **سلة النظام** بعد اكتمال الروابط الواردة، كي يعرض مربع التأكيد اسم الملف ومساره وعدد الملفات التي تشير إليه وتحذيرًا أقوى إن كان متعقبًا في Git. يكون الإلغاء هو الإجراء الافتراضي، ولا يوجد حذف نهائي أو رجوع صامت إلى `unlink`.
- [x] بعد النقل: يزال الملف من الشجرة وفهرس البحث، ويفتح أقرب ملف مناسب إن كان الملف النشط، وتبقى الملاحظات وسجل القراءة محفوظين بحالة «ملف مفقود» ولا يُحذفان تلقائيًا.
- عدم إضافة الرسوم الشبكية أو علامات التبويب المتعددة أو تقسيم النوافذ في هذه المرحلة؛ التاريخ والمعاينة يحلان حاجة القارئ بتعقيد أقل.

### 3. إضافة «ذاكرة القارئ»

- [x] حفظ آخر ملف وموضع القراءة لكل مجلد وملف، مع استعادة الموضع باستخدام أقرب عنوان ونسبة التقدم داخل القسم، ثم النسبة العامة كخيار احتياطي.
- [x] إضافة مفضلة للملف أو العنوان أو مقطع محدد، وقائمة «أكمل لاحقًا».
- [x] دعم تمييز النص وإضافة ملاحظة جانبية دون تغيير Markdown. ترتكز الملاحظة إلى النص المحدد والسياق السابق واللاحق وأقرب عنوان.
- [x] عند تغير الملف، يعاد تثبيت التمييز بالسياق؛ إذا تعذر ذلك يظهر كتعليق غير مرتبط يحتاج مراجعة، ولا يُحذف تلقائيًا.
- [x] إتاحة تصدير جميع المفضلات والتعليقات إلى Markdown، مع رابط الملف والعنوان والنص المقتبس.
- [x] إظهار الملفات التي تغيرت منذ آخر قراءة اعتمادًا على `mtime` والمحتوى المتحقق منه، دون بناء نظام فروق كامل في الإصدار الأول.
- [x] حفظ الحالة خارج مجلد المستخدم: `Application Support/Matn` على macOS، و`XDG_DATA_HOME/matn` على Linux، و`APPDATA/Matn` على Windows. تُكتب ملفات JSON ذريًا عبر ملف مؤقت ثم إعادة تسمية.

### 4. تعميق تجربة القراءة

- [x] إضافة وضع تركيز يخفي شريط الأدوات أثناء القراءة ويعيده عند الاقتراب من الحافة أو استخدام لوحة المفاتيح.
- [x] تنظيم الشريط الجانبي إلى: الملفات، والبحث، والمفضلة، والروابط الواردة، والتعليقات؛ ويخرج فهرس العناوين النصي الحالي منه.
- [x] استبدال فهرس العناوين الحالي بـ **خريطة مستند مصغّرة** شبيهة بنمط ChatGPT: مسار رأسي هادئ بمحاذاة حافة مساحة القراءة، تمثل علاماته عناوين `h1–h3` ويتميز القسم الحالي بصريًا. هذه خريطة تنقّل وليست شريط تمرير ثانٍ.
- [x] عند المرور على الخريطة أو تركيزها بلوحة المفاتيح، تظهر بطاقة قريبة تعرض العناوين المحيطة بالموضع مع إبراز العنوان الحالي؛ يؤدي اختيار عنوان إلى الانتقال إليه وتحديث الرابط. لا تعتمد الوظيفة على المرور وحده: يوجد زر واضح يفتح قائمة العناوين باللمس ولوحة المفاتيح.
- [x] تتكيف كثافة الخريطة مع المستندات الطويلة بتجميع العلامات المتقاربة مع الحفاظ على مستويات العناوين، وتختفي عندما لا يملك المستند بنية مفيدة. في الشاشات الصغيرة تتحول إلى زر يفتح لوحة عناوين، ولا تزاحم النص أو شريط تمرير النظام.
- [x] إضافة إعدادات مسبقة: «مقالة»، و«بحث أكاديمي»، و«توثيق برمجي»، و«نص طويل»، مع إمكانية تخصيصها.
- [x] تحسين العرض للشاشات الصغيرة بدل إخفاء الشريط الجانبي كليًا؛ يتحول إلى لوحة قابلة للفتح.
- [x] إضافة طي الأقسام، ونسخ رابط إلى عنوان، ونسخ مقتطف منسق.
- تأجيل العرض التقديمي وTTS والمزامنة السحابية حتى تثبت بيانات الاستخدام وجود حاجة حقيقية.

## الواجهات والبيانات

- [x] إضافة خيار CLI باسم `--editor <executable>` ومتغير `MATN_EDITOR`. يُشغّل الملف كوسيط مباشر باستخدام `spawn` و`shell:false`؛ لا تقبل أوامر shell.
- [x] إضافة خيار صريح `--allow-file-actions` لتفعيل إجراءات إدارة الملفات. يبقى النقل إلى السلة معطلًا إن لم تتوفر آلية سلة آمنة على نظام التشغيل، ولا يتحول إلى حذف دائم.
- إضافة API محلية:
  - `GET /api/search` للبحث في المجلد.
  - `GET /api/context` للروابط الصادرة والواردة وبيانات الملف.
  - `GET/PUT /api/state` لحالة القراءة والمفضلة والتعليقات.
  - `POST /api/open-editor` لفتح الملف في المحرر المحدد.
  - `POST /api/trash` لنقل ملف Markdown داخل جذر القراءة إلى سلة النظام فقط.
- تتطلب عمليات `PUT/POST` رمز جلسة وتحقق `Host` و`Origin`، وتقبل مسارات داخل جذر القراءة فقط.
- تنفذ سلة النظام عبر محولات صغيرة لكل من macOS وWindows وLinux باستخدام `spawn` و`shell:false`، مع منع المجلدات والروابط الرمزية الخارجة عن الجذر والملفات غير المدعومة.
- مخطط التعليق: معرّف، ومسار نسبي، والنص المطابق، والسياق السابق واللاحق، والعنوان الأقرب، واللون، والملاحظة، وتواريخ الإنشاء والتحديث.
- يبنى فهرس البحث في الذاكرة بعد فتح المجلد دون تعطيل عرض الملف الأول، ويحدّث الملفات المتغيرة فقط.

## تكامل Claude Code

- [x] إضافة أمر ثابت `matn <path>` يفتح الملف أو المجلد في نافذة «متن» الحالية أو يعيد استخدامها.
- [x] توفير Skill على مستوى المشروع باسم `/matn-open` تستدعي المشغّل المنفصل صراحة؛ يؤجل تغليفها كإضافة عامة وأداة MCP حتى تظهر حاجة إلى توزيع خارج المشروع.
- [x] لا يضاف Hook يفتح كل ملف Markdown يقرؤه Claude Code، لأنه سيسرق التركيز عند قراءة ملفات التعليمات والتوثيق الداخلية.
- [x] يؤجل عرض «متن» داخل تبويب VS Code؛ فهو يحتاج امتداد Webview أو Custom Editor مستقلًا، وليس إعدادًا داخل Claude Code، ولا يبرر تكلفته قبل إثبات استخدام التكامل الخارجي.

## الاختبارات ومعايير القبول

- اختبارات وحدة لنواة العرض، وتطبيع البحث العربي، وحل الروابط والأسماء البديلة، وإعادة تثبيت التعليقات.
- اختبارات أمنية لعبور المسارات، وDNS rebinding، وCSRF، والروابط غير الآمنة، ومحاولات فتح محرر أو نقل ملف خارج الجذر، ومنع المجلدات والروابط الرمزية والهروب عبر المسار الحقيقي.
- اختبارات تكامل لحفظ الحالة ذريًا، وتلف ملف الحالة، وإعادة التشغيل، ونقل ملف إلى سلة النظام أو تعذرها، وتغير المحتوى أثناء القراءة؛ ويثبت الاختبار أن المسار لا يستدعي حذفًا دائمًا.
- اختبارات Playwright لمسارات: فتح مجلد، والبحث، والانتقال عبر رابط، والرجوع، وحفظ موضع القراءة، وإضافة تعليق، وفتح المحرر الخارجي، والتنقل بخريطة المستند، وإلغاء نقل الملف ثم تأكيده.
- اختبارات وصول لخريطة المستند ومربع تأكيد النقل: ترتيب التركيز، وأسماء الأزرار، والاستخدام بلوحة المفاتيح واللمس، وعدم حبس المستخدم داخل البطاقة المؤقتة.
- مجموعة مستندات عربية وثنائية الاتجاه تشمل العناوين التي تبدأ بالإنجليزية، والجداول، والقوائم، والحواشي، والرياضيات، وMermaid، والكود.
- القبول يتطلب:
  - عدم تعديل محتوى أي ملف Markdown في جميع المسارات؛ ولا ينقل الملف نفسه إلا بعد إجراء صريح ومؤكد إلى سلة نظام قابلة للاستعادة.
  - عرض الملف الأول قبل اكتمال فهرسة المجلد.
  - ظهور نتائج البحث خلال 200ms بعد الفهرسة على مجموعة مرجعية من 2,000 ملف.
  - استعادة موضع القراءة ضمن القسم الصحيح بعد تعديلات طفيفة أعلى المستند.
  - استمرار العمل كاملًا دون اتصال أو حساب.

## الافتراضات والحدود

- المنتج محلي خفيف، يعمل عبر Node والمتصفح، وموجّه أولًا لسطح المكتب.
- تبقى تبعيات التشغيل npm صفرًا؛ تبعيات التطوير مسموحة للاختبار فقط.
- لا محرر مدمج، ولا AI، ولا تعاون، ولا مزامنة، ولا قواعد بيانات، ولا منظومة إضافات داخل «متن» في الخارطة الحالية؛ إضافة Claude Code تكامل خارجي محدود.
- الأولوية التنافسية: التنقّل بين الملفات، وذاكرة القراءة، والتعليقات غير المتلفة، والبحث العربي؛ التصدير والسمات الحالية تبقى مزايا داعمة.
